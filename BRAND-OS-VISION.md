You aren't building **just a YouTube channel**. You're building a **media company**. Every AI tool should be trained on the same standards so everything — from thumbnails to websites to mobile apps — looks like it came from one creative team.

### 1. Editorial Standards
Every claim must have a source. Primary source > News article > Social media. Never use AI-generated facts. Always separate FACT / ANALYSIS / SPECULATION. Never exaggerate. Never clickbait beyond what the evidence supports.

### 2. Thumbnail Bible
Headline 3–5 words. Robot: large subject, 30–40% of frame. Background minimal. Color mostly neutral, orange accent only. Text never outlined. No emojis, arrows, circles, shocked faces. Eventually 50+ thumbnail examples.

### 3. Video Bible
Cold Open → Question → Evidence → Analysis → Verdict → Future Outlook.

### 4. B-Roll Library
Reusable owned shots: robot walking, robot charging, warehouse, kitchen, office, factory, PCB closeup, servo motor, circuit board, AI chip, GPU, satellite, drone, server rack. Over a year: thousands of reusable shots.

### 5. Motion Rules
Camera speed, lens choice, animation easing, graph style, chart style, zoom amount, fade duration. Nobody documents this. Studios do.

### 6. AI Prompt Library
One prompt for every task: thumbnail, chart, Veo, Higgsfield, Midjourney, image, infographic, diagram, website, UI.

### 7. Story Skeleton Library
A: Question · B: Timeline · C: Problem→Solution · D: Investigation · E: Day in the Life · F: Economics · G: Security Analysis · H: Engineering Deep Dive. (Feeds C-3 structural variance directly.)

### Long-term production system
Research Agent → Knowledge Base → Script Agent → **Human Pass** → Storyboard Agent → Image Agent → Video Agent → Voice Agent → Editor Agent → Thumbnail Agent → Publishing Agent.

Scales from one channel to a media network: a second channel (AI software), third (EVs), fourth (home automation) inherit the production engine while keeping their own editorial identity.
